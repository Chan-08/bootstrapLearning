Bootstrap Classes & Techniques Used

This guide explains the Bootstrap utilities and components used in the demo portfolio. No custom CSS is required.

Layout & Containers

The site uses .container to center content and apply responsive paddings. Spacing uses .py-*, .my-*, and .g-* utilities.

  • .container, .row, .col-* for the grid
  • .row-cols-* to auto-size columns per breakpoint
  • .g-* for gutters; .mb-*, .mt-*, .py-* for spacing
  • .bg-light, .bg-body-tertiary, .border-bottom for section styling
Tip: Use responsive classes like .row-cols-lg-3 to change layouts across breakpoints.

Navbar

Built with .navbar, .navbar-expand-lg, .navbar-toggler, and a collapsible .navbar-collapse.

  • .sticky-top keeps the navbar fixed within normal flow
  • .bg-body-tertiary + .border-bottom provide subtle separation
  • Buttons use .btn variants: .btn-primary, .btn-outline-secondary

Hero & Utilities

Typography uses .display-*, .lead, and weight classes like .fw-bold. Media uses .ratio for responsive aspect ratios.

  • .badge and .text-bg-* for labeled chips
  • .ratio .ratio-16x9 to maintain a 16:9 image frame
  • .object-fit-cover to crop images within containers
  • .d-flex, .gap-2, .flex-wrap to control layout spacing

Cards & Grid

Projects use the card component with headers, footers, and images. The grid is responsive via .row-cols-*.

Accordion (FAQ)

Built with the .accordion component, driven by collapse JavaScript included in the Bootstrap bundle.

Each accordion body is a collapse target toggled by an associated button using data-bs-target.

Forms & Validation

Uses .form-floating for floating labels, .input-group for prefixed inputs, and the custom validation styles with .was-validated.
